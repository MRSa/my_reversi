
以下の内容に従い、人間 vs コンピューター対戦のリバーシゲームを生成したいと考えています。

■開発環境
- 開発環境は、Windows です。
- アプリケーションのターゲットは、 Windows と Web を想定してください。 
- flutter のプロジェクトが存在しない場合は、新規作成をお願いします。プロジェクトが存在する場合は、当該プロジェクトを改造してください。
- プロジェクトのルートディレクトリ以下に成果物を生成してください。

■ 基本フロー
1. 【側面選択画面】：黒（先攻）or 白（後攻）を選択可能。「先攻＝人間が黒」「後攻＝人間が白」とし、選択後に盤面画面へ遷移。
2. 【ゲーム開始】：画面遷移と同時にタイマーと石の配置を初期化。人間側から先手開始。
3. 【ゲーム終了】：全マス埋まるかパス継続でゲームオーバー。勝敗表示（例：「黒の勝利」「白（あなた）の勝利」など）。
4. 【結果画面】：勝敗メッセージ＋「再戦する」「終了する」ボタン。再戦は側面選択画面へ、終了はアプリ終了またはホーム画面へ遷移。

■ タイマー機能（必須）
- 各ターンの持ち時間を60秒に設定。
- UI上部に進行状況を示すプログレスバーを表示（残時間が少なくなるにつれて色が変わる：緑→黄→赤）。
- ターン開始時にタイマーをリセット・カウントダウン開始。
- 残り時間0秒になった場合、自動で合法な場所に石を配置する（複数ある場合は、中心or角・辺に近い位置を優先する簡易ヒューリスティック適用）。
- パスが発生した場合は相手のターンに切り替わり、相手のタイマーが同時に開始される。

■ コンピューターAI（簡易実装で可）
- 合法手をすべて生成し、以下の評価関数でスコアリングして最高スコアの手に配置する：
  ・角の優先度：＋100
  ・辺（角以外の端）：＋20
  ・隣接マスの安定性：＋5～10
  ・全体の差：±1
- αβ法やミニマックスは省略可（ボードサイズ8×8のため全手探索でも実用範囲）。

■ 技術要件
- フレームワーク：Flutter (Dart) latest stable
- 状態管理：riverpod または provider（選択し理由をコメントで明記）
- ルーティング：Navigator 2.0 または go_router（画面遷移の設計思想を簡潔に記述）
- ゲームロジックとUIを分離（GameBoardState/Logic クラスとUIウィジェットを分ける）
- おきられないターンは「パス」ボタンを表示し、クリックでターン交代
- 石の配置アニメーション：simple `AnimatedContainer` or `Hero` で滑らかに
- コードは読みやすく、ロジック部分（GameLogic）とUI部分（GameScreen）を分離して構成してください

■ UI/UX 要件
- 盤面：8×8グリッド、中央寄せ、レスポンシブ対応（縦横どちらでも操作可能）
- 石：黒（#000000）、白（#FFFFFF）、枠線（#333333）、角丸円形
- タップ可能な合法マスのみをハイライト（例：半透明の点や輪郭線）
- タイマーバーは画面最上部に配置。残り時間が10秒以下で振動または赤点滅UIを推奨（ビルトインのみで実装可ならOK）
- 結果画面はモーダルまたはフルスクリーンオーバーレイで表示

■ 出力形式
- 1つのファイル `main.dart` で完結するか、必要な場合は明確なファイル構成（lib/以下）を記載し、各ファイルの内容を完全なコードとして提示してください。
- コードには日本語コメントを随所に追加し、依存関係（pubspec.yamlの追加分など）も明示。
- 動作検証済みの実装を想定し、エラーハンドリング・境界条件（パス連鎖・時間切れ連続など）にも対応したロジックを含めること。

上記の実装の第一段階として、まず、以下の仕様に基づいた基本機能の計画・実装をお願いします。

**【基本仕様】**
1. 8x8のグリッド盤面を表示すること。
2. 黒と白の石を配置でき、リバーシのルール（挟んだ石をひっくり返す）に従って動作すること。
3. プレイヤーが交互に打つターン制であること。
4. 石が置ける場所があるか判定し、置けない場合は自動的にパスすること。
5. 両者ともに打てる場所がなくなった、または盤面が埋まった時にゲーム終了となること。

**【技術的要望】**
- 状態管理はシンプルに `setState` または `Provider/Riverpod` など、適切と思われる方法で実装してください。
- コードは読みやすく、ロジック部分（GameLogic）とUI部分（GameScreen）を分離して構成してください。
- まずは人間vs人間の対戦形式で実装し、後のステップでCPU対戦を追加します。

**【追加仕様】**
1. **持ち時間バーの表示：** 画面上部に、現在のターンの残り時間を視覚的に示すプログレスバー（LinearProgressIndicatorなど）を表示してください。
2. **タイマー処理：** 1手につき制限時間（例：15秒）を設けてください。ターンが回ってきた瞬間からカウントダウンを開始します。
3. **自動着手機能：** 制限時間がゼロになった場合、以下の処理を自動で実行してください。
   - 現在のプレイヤー（人間またはCPU）が置ける場所があるか確認する。
   - 置ける場所がある場合、その中からランダムに1箇所を選んで自動的に石を置く。
   - 自動着手後、すぐに次のターンへ移行する。
4. **タイマーの停止：** 石を置いた瞬間、またはパスが発生した瞬間にタイマーをリセットし、次のプレイヤーのカウントダウンを開始してください。
